# C38 · Crunch Scale — Syllabus

**Format:** 12 weeks · self-paced · open-source (GPL-3.0). Each week = 3 lectures, 3 exercises, 2 challenges, a mini-project, homework, and a quiz.

**Prerequisites:** Comfort with SQL at the level of [C33 Crunch SQL](../C33-CRUNCH-SQL/) — joins, aggregation, CTEs, and window functions — plus basic Python. No prior growth, marketing, or finance background assumed. **Data lives in SQL (PostgreSQL/SQLite) and pandas, never in a spreadsheet.**

**Assessment (honor-based):** tick each lesson complete in the reader; finish the weekly mini-project and quiz. A certificate is issued on 100% completion to One-Stop members.

| Week | Theme | Mini-project |
|------|-------|--------------|
| 1 | Growth foundations — north-star, AARRR, event instrumentation into a warehouse | Instrument an event schema + build a metrics baseline |
| 2 | Acquisition funnels + multi-touch attribution in SQL | Build a channel funnel + attribution report |
| 3 | Activation, time-to-value, onboarding-funnel diagnosis | Diagnose + fix a leaky onboarding funnel |
| 4 | Retention + cohort analysis + survival curves | Build a cohort-retention table + curve |
| 5 | LTV, CAC, payback, contribution margin, unit economics | Model unit economics for a subscription business |
| 6 | RevOps + the customer data stack (events → staging → marts) | Model a warehouse mart layer as one source of truth |
| 7 | Segmentation — RFM, behavioral, clustering-based intelligence | Build and profile a customer-segment model |
| 8 | Experimentation — A/B design, sizing, significance, traps | Design + analyze an A/B test end to end |
| 9 | Pricing + packaging + price sensitivity + tiering | Model a pricing change and forecast its revenue impact |
| 10 | Lifecycle + AI personalization — churn/propensity, next-best-action | Score churn + build next-best-action logic |
| 11 | Forecasting growth — time series, scenarios, revenue projection | Build a revenue forecast + scenario model |
| 12 | Capstone — end-to-end growth + revenue system on a warehouse | Ship a full growth system with metrics, tests, and forecast |

**Outcome:** run a company's growth on evidence — instrument, measure, experiment, price, and forecast a revenue system entirely in SQL and pandas.
